## Dogfood walk-through

Walked the 7 T2 skills in lifecycle order plus the T1 prelude, checking that skill N's `## Output` / `## Handoff` matches skill N+1's `## Input`. The chain is coherent end-to-end:

- **brainstorm** → writes `.guild/spec/<slug>.md`; handoff invokes `guild:team-compose` with the spec path.
- **team-compose** → `## Input` consumes `.guild/spec/<slug>.md` (matches brainstorm output); writes `.guild/team/<slug>.yaml`; handoff invokes `guild:plan` with the team.yaml path.
- **plan** → `## Input` requires both `.guild/spec/<slug>.md` and `.guild/team/<slug>.yaml` (matches team-compose output); writes `.guild/plan/<slug>.md` with frontmatter `approved: false` until user approves, then `approved: true`; handoff invokes `guild:execute-plan`.
- **execute-plan** → `## Input` requires the approved plan plus `team.yaml`, creates `.guild/runs/<run-id>/`; per-lane flow invokes `guild:context-assemble` (context bundle → dispatch → receipt); aggregates `.guild/runs/<run-id>/assumptions.md`; handoff invokes `guild:review` with `run_id`, `handoffs_dir`, `plan_path`, `lane_count`, `backend`.
- **context-assemble** → invoked per-lane by execute-plan; writes `.guild/context/<run-id>/<specialist>-<task-id>.md`; each receipt names `bundle_path`, `token_estimate`, `layers_included` — execute-plan reads those before dispatching.
- **review** → `## Input` reads `.guild/runs/<run-id>/handoffs/*.md` (matches execute-plan's stop condition) plus spec and approved plan; writes `.guild/runs/<run-id>/review.md`; handoff invokes `guild:verify-done` with `run_id`, `review_path`, `handoffs_dir`, `plan_path`, `blockers`, `followups`.
- **verify-done** → `## Input` requires `review.md`, spec `success_criteria`, and `assumptions.md` (all produced upstream); runs five checks; writes `.guild/runs/<run-id>/verify.md`; handoff targets `guild:reflect` (forward reference into P5, explicitly acknowledged — skill instructs "if guild:reflect is not yet installed, stop here and return verify.md path to user", so the forward ref is not a gate blocker).

The T1 **guild:principles** is the mandatory specialist prelude per `guild-plan.md §2` + `§6.4`; it has no Input/Output/Handoff chain (it's a reference skill loaded by every specialist), so it correctly sits outside the lifecycle DAG.

Artifact-path contract is consistent across the chain: `.guild/spec/<slug>.md` → `.guild/team/<slug>.yaml` → `.guild/plan/<slug>.md` → `.guild/runs/<run-id>/{handoffs/*, context/*, assumptions.md, review.md, verify.md}`. No drift detected between producer and consumer paths. The context-bundle token contract (~3k target, 6k hard cap) is stated in context-assemble's `## Size budget` section — this satisfies the P1 plan-row criterion "verify context bundles under 3k tokens" at the specification level (runtime enforcement lands when bundles are actually written during a live `/guild` run).
