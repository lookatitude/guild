### Findings

#### F5-T8 — Token-shape group misses the GitHub token families and `github_pat_` fine-grained PATs (medium)

The bundle's known-shape list explicitly names: `sk-ant-`, `sk-`, `xoxb-`,
`AKIA*`, `github_pat_`, `ghp_`, `ghs_`, JWT `eyJ...`. The current
`TOKEN_SHAPE_PATTERNS` (`benchmark/src/redact-log.ts:67-75`) covers:
- `sk-(ant-)?` ✓
- `ghp_` ✓
- `xox[bp]-` ✓
- `AKIA*` ✓
- `eyJ.eyJ.sig` ✓
- `Authorization: Bearer …` ✓
- `\bBearer …{16,}` ✓

**Missing:**
- `ghs_<36>` — GitHub server-to-server tokens (named in the bundle).
- `gho_<36>` — GitHub OAuth tokens.
- `ghu_<36>` — GitHub user-to-server tokens.
- `ghr_<36>` — GitHub refresh tokens.
- `github_pat_<22>_<59>` — GitHub fine-grained personal access tokens (named in the bundle).

**Vector.** A tool error (e.g., `git push` over HTTPS using a `ghs_*` server
token, a `github_pat_*` fine-grained PAT echoed in a `gh auth status`
output, or a `gho_*` from an OAuth-flow stderr) lands in
`result_excerpt_redacted`. The high-entropy group (group 4) catches some
shapes (≥20 chars base64-like) — but the prefix gives the secret its
shape and the `_` separator in `github_pat_` plus the underscore in the
ghs/gho/ghu/ghr family are NOT in the base64 alphabet
(`[A-Za-z0-9+/=]{20,}`), so the high-entropy group **does not catch them
in full** (it would catch only the post-`_` segment).

Concrete repro: `redactField("running with ghs_aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa now")` →
output retains `ghs_…` (the 36-char alphanumeric body matches the
high-entropy detector inside the body but the prefix `ghs_` is preserved,
so the *type* of the secret is leaked AND the whitelist for run-id /
commit context fires before the entropy match).

**T8-trigger reason.** The drift sentinel
(`tests/redact-log.test.ts:409-421`) requires
`TOKEN_SHAPE_PATTERNS` ≡ `runner.ts:182-193 REDACTION_PATTERNS`. Patching
`redact-log.ts` alone breaks the drift test; patching `runner.ts` is
outside the security carve-out (lane block §"forbidden": "writing
application code outside `redact-log.ts` and its tests"). Backend (T8)
must extend BOTH files in lockstep.

**Required patches (specification for T8).**

In BOTH `benchmark/src/runner.ts` (insert into `REDACTION_PATTERNS`
between the `ghp_` and `xox[bp]-` rows) AND
`benchmark/src/redact-log.ts` `TOKEN_SHAPE_PATTERNS` (verbatim mirror):

```ts
{ re: /\bgh[suor]_[A-Za-z0-9]{36}\b/g, shape: "github-token" },
{ re: /\bgithub_pat_[A-Za-z0-9_]{82}\b/g, shape: "github-fine-pat" },
```

(`gh[suor]_` = ghs / ghu / gho / ghr unified into one regex; `ghp_` is
already covered above. The fine-grained PAT shape is `github_pat_` +
22-char prefix + `_` + 59-char body = 82 alphanumeric+underscore chars
after the `github_pat_` literal.)

**Tests T8 must add to `benchmark/tests/redact-log-patterns.test.ts`:**

1. `redacts a GitHub server-to-server token (ghs_…)` — input `ghs_<36>`, expect `TOKEN_REDACTED`.
2. `redacts a GitHub OAuth token (gho_…)` — input `gho_<36>`, expect `TOKEN_REDACTED`.
3. `redacts a GitHub user-to-server token (ghu_…)` — input `ghu_<36>`, expect `TOKEN_REDACTED`.
4. `redacts a GitHub refresh token (ghr_…)` — input `ghr_<36>`, expect `TOKEN_REDACTED`.
5. `redacts a GitHub fine-grained PAT (github_pat_…_…)` — input `github_pat_<22>_<59>`, expect `TOKEN_REDACTED`.
6. `does not redact 'ghs_short'` — negative case (less than 36 chars after prefix).

- Re-run `npx vitest run tests/redact-log.test.ts tests/redact-log-patterns.test.ts` and confirm all pass (drift sentinel still green; new tests green).

#### F5-LOW-1 — High-entropy whitelist scope too generous on bare 40/64-hex (low)

`isWhitelistedHighEntropy` (`benchmark/src/redact-log.ts:189-196`)
whitelists ANY 40-char or 64-char hex string globally, with no context
gate. The comment explains the carve-out as "common case of bare hashes
in test names / file lists". This means a 40-char hex API hash (e.g., a
SHA-1-shaped bearer token) leaks unredacted.

**Vector.** A field containing `auth=a3b2c1d0e9f8a7b6c5d4e3f2a1b0c9d8e7f6a5b4`
gets group 3 (`auth=…`) match → kv-redacts. Good. But a field
containing only `a3b2c1d0e9f8a7b6c5d4e3f2a1b0c9d8e7f6a5b4` with no
key-prefix passes through. The architect's rationale is documented; this
is a CONSCIOUS tradeoff. Not a finding to fix — record as documented
limitation.

**Disposition.** Documented limitation. NOT filed to FOLLOWUPS (architect
contract; downgrading would create false-positive noise). Surface in this
doc for posterity.
